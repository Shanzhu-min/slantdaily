
## 需求描述
我要创建一个逻辑游戏站，主题是slant的每日游戏。页面包括：
- Daily Challenge: 站点的index页面，页面内容设计图参考newFiles/icon/daily challenge.png;页面的设计需求在newFiles/design/daily challenge.md ;页面的文字内容在newFiles/json/daily challenge.json中；
- archive: 页面内容设计图参考newFiles/icon/archive.jpg;页面的设计需求在newFiles/design/archive.md ;页面的文字内容在newFiles/json/archive.json中；
- achievement: 页面内容设计图参考newFiles/icon/achievement.jpg;页面的设计需求在newFiles/design/achievement.md ;页面的文字内容在newFiles/json/achievement.json中；
- printable slant: 页面内容设计图参考newFiles/icon/printable slant.jpg;页面的设计需求在newFiles/design/printable slant.md ;页面的文字内容在newFiles/json/printable slant.json中；
- Rules: 页面内容设计图参考newFiles/icon/rules.jpg;页面的设计需求在newFiles/design/rules.md ;页面的文字内容在newFiles/json/rules.json中；
- 导航页文案原型参考newFiles/design/nav.md；项目运行成功后，导航文案也迁移到 src/locales/ 下，运行时不再依赖 newFiles 中的内容。


##设计约束
- 全站的H1、H2、P标签等的字体、字号需要保持一致。
- Daily Challenge、archive、achievement、printable slant、Rules 的原型文案从 newFiles/json 中读取；项目运行成功后，将所有页面文案与导航文案迁移到 src/locales/ 下，运行时不得 import/read/link newFiles/*。文章内容用 MDX 实现，支持后续增加多语言。
- newFiles/ 只作为原型输入材料。图片素材在实现时迁移到 public/images/ 下，页面运行时统一使用 /images/... 路径，不依赖 newFiles/。
- 在设计时，要考虑站点的可配置性。即后续可以通过对json文件的替换，实现站内所有内容更换。
- CSS都写入到globals.css中。避免各页面中维护自己的CSS配置，从而导致将来维护时的投入成本与修改风险。
- 全站是英文站，不允许出现中文字符。
- 全站中比较配套的颜色风格为：背景色为#F8FAFC；H1、H2的字体颜色为#1E3A8A；正文的字体色为#6E7C8C；内容框中的底色为#FFFFFF；主CTA的背景色为#5A7bc0,字体色为#FFFFFF；副CTA的背景色为#FFFFFF，字体色为#325AAF；把它们定义为CSS变量，以便后续修改。

## 技术约束

### 技术栈
- Next.js 15 App Router + TypeScript 严格模式
- 国际化：next-intl v4.x
- MDX：@next/mdx + 动态 import() 直接导入 .mdx 文件（不使用 next-mdx-remote）
- 样式：Tailwind CSS + shadcn/ui + lucide-react

```mdx
内容正文...
```

- 内容目录支持嵌套：`content/en/[type]/[subdir]/[slug].mdx`
- URL：`/[locale]/[contentType]`（列表页）、`/[locale]/[contentType]/[slug]`（详情页）

### 路由规则

- Daily Challenge 是首页，固定路由为 `/`。
- archive / achievement / printable slant / rules 都有独立页面设计，写成固定路由：`/archive`、`/achievements`、`/printable-slant`、`/rules`。
- 使用 next-intl 的 `localePrefix: "as-needed"`。默认英文站不显示 `/en` 前缀；后续增加非默认语言时，非默认语言使用 `/{locale}/...`。
- `[...slug]/page.tsx` 只用于后续 MDX 文章详情或列表页，不承载以上 5 个核心固定页面。

---

### SEO 要求

- `[locale]/layout.tsx` 的 generateMetadata：og:image / twitter:image 用 `${NEXT_PUBLIC_SITE_URL}/images/hero.webp` 绝对路径
- Organization JSON-LD 必填：logo、image、url、name
- 文章页 generateMetadata：title、description、alternates.languages（hreflang）、OpenGraph
- 文章页 JSON-LD：内联 `Article` + `BreadcrumbList` 对象（通过通用 `<JsonLd>` 组件注入）
- 列表页 JSON-LD：内联 `ItemList` 对象
- sitemap.ts 用 `NEXT_PUBLIC_SITE_URL` 环境变量，不硬编码域名
- **sitemap 生成规则（重要）**：必须通过 `getAllContentPaths()` 扫描 content 目录下实际存在的 `.mdx` 文件来生成 URL，再加上首页、列表页、法律页等静态页面。**禁止**从硬编码数据源（如 `BOSS_CARDS` 数组、手写 URL 列表、navigation config 的固定路径）生成 sitemap 条目。原因：列表页展示的条目不一定都有对应 MDX 文件（如 28 个 boss 只有 1 个写了攻略），硬编码会产生大量 404/500 页面的 sitemap 条目
- robots.ts 动态生成，包含 sitemap 链接
- 每页有且仅有一个 H1。固定页面组件从 src/locales/ 的页面 JSON 中读取并渲染唯一 H1；MDX 文章页由 DetailPage 自动用 metadata.title 渲染 H1，MDX 正文不写 H1。
- alternates.languages 与 routing.ts locales 一致

---

### 样式规范

- 颜色使用 `--color-*` CSS 语义变量，禁止 TSX 中硬编码颜色（`#xxxxxx`、`bg-red-500`、`rgba(...)`）。后续换站点主题时，优先只修改 `src/app/globals.css` 中的 `:root` 变量。
- 图标用 lucide-react，禁止 emoji；同一页面内卡片图标不重复
