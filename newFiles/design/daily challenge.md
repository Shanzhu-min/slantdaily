# 背景
- 提供slant的在线游戏，主要覆盖daily模式的场景，以便和其它游戏站型成差异。
- 该页面的框架和元素固定，但是内容是从外部的json中引入 。

# 页面元素
- 游戏区：游戏区通过iframe的方式引入。方便将来游戏内容和页面文案的分别维护工作。关于iframe URL、尺寸、fallback、sandbox 策略等，你来设计 。
- sections：由若干个section组成（section的数量不固定，但是样式一致），每个section中，包括一个标题（H2）和一段文案描述。section中标题和内容的文案，来源于外部json文件。
- FAQ：FaQ区域由若干个问题和答案组成，问题和答案的文案，来源于外部JSON文件。

# 设计要求
- 对移动端支持友好。

# 游戏区域
- 在当前阶段，通过从iframe的方式引入后，在页面中显示“slant daily”的字样即可。后续再对游戏页面进行设计和优化。
